## 프로젝트 한눈에 보기
- **목표**: 한국에서 검증된 닥터나우를 일본 법규·문화·사용자 기대에 맞춰 **신뢰 가능한 서비스**로 재구축
- **범위**: 환자 앱 · 병원 웹 · 약국 웹 **3자 서비스 전면 재설계 + 마케팅/퍼널 실험 운영체계**
- **기여도**: 기획 50% / UI·UX 90% / 실행 리드 (PM + Product Design)
- **출력물**: Android, iOS, Mobile Web 전 채널 출시 및 운영 가이드

## 배경과 문제 정의
1. **법·규제 차이** — 화상진료·복약지도 필수, 약 배송 허용 등으로 한국 플로우와 완전히 다른 5-Step 진료 여정 필요
2. **로컬 UX 불일치** — 일본 사용자에게 "외국 앱 같다"는 피드백 반복 → 신뢰도 저하, 복잡한 카피/레이아웃로 전달 실패
3. **예약 니즈 차이** — 한국은 "지금 당장", 일본은 "내 일정에 맞게" 예약 → 오늘/내일 2슬롯 UI에서 대규모 이탈(증상→시간 선택 구간 38.3%)
4. **운영/신규 기능 요구** — 약 배송 추적, 의사-환자 채팅, 일본식 로그인 등 리걸·CS 이슈를 해결할 기능 부재

## 해결 전략과 실행
### 1. 규제 대응형 5-Step 진료 플로우
- **홈 → 예약 → 화상진료 → 배송방법/약국 선택 → 복약지도/약수령**으로 필수 단계만 남긴 여정 설계
- 병원·약국·유저 3자 상태를 한눈에 볼 수 있는 **Status 기반 UI** 설계로 운영 혼선을 최소화

### 2. 일본식 비주얼 & 카피 시스템
- 일본 경쟁 서비스 리서치 후 **Hiragino Kaku Gothic** 폰트, 조밀한 레이아웃, 큰 CTA, 섹션 설명 텍스트 등 현지 UX 문법 적용
- 의사 인터뷰로 수집한 용어·증상 카테고리를 기반으로 **증상 중심 정보 구조**로 전면 개편 → "타사보다 익숙하다"는 정성 피드백 확보

### 3. 예약/매칭 퍼널 실험
- **캘린더형 슬롯 UI** 도입, 3일 이후 예약 비중 40% 이상 확보 & 전환율 9.43% → 12.5%(+3.07pp)
- "가장 빠르게 진료받기" 자동매칭 플로우 설계 → 의사·시간 선택 없이 매칭, 문제 구간 이탈 38.3% → 34.3%, 진료신청 완료율 +6.3pp

### 4. 온보딩 & 신규 기능 우선순위화
- 로그인/주소 권한 이탈 데이터를 근거로 **E-mail 로그인** 추가 → 전체 가입의 29%, 계정 관련 CS 25건/월 → 0~1건/월(80%↓)
- RICE 점수로 "이메일 로그인 → 의사-환자 채팅 → 배송 트래킹" 순으로 합의, 양측(앱/웹) UX를 동시에 설계
- 일본 음식 배달 컨벤션을 차용한 **배송 현황 추적** UI + 약사웹 집하 프로세스 → 배송 CS 80% 감소

## 나의 역할
- **Research & Localization**: 사용자·의사 인터뷰, 경쟁 리서치, 카피/증상 구조 정의
- **Experiment Design**: 주요 퍼널별 가설 수립 → 캘린더 실험, 자동매칭, 로그인 지연 실험 설계
- **Execution Leadership**: 디자인 시스템, 제품 요구사항, 우선순위(OKR & RICE) 정렬, 데이터 리뷰 주도
- **Cross-functional Alignment**: 마케팅 CAC/CPA 지표와 제품 퍼널 지표를 일원화해 주간 리뷰 운영

## 배운 점
- UX 로컬라이징은 시각 요소뿐 아니라 **법적 행위 순서, 인터랙션 컨벤션, 카피 톤**이 동시에 맞아야 신뢰를 얻을 수 있음
- 퍼널 실험은 정량(Amplitude)과 정성(인터뷰)을 번갈아야 빠르게 가설을 정제할 수 있음
- 로그인/권한과 같은 필수 단계도 **시점 재배치**를 통해 체감 난도를 낮출 수 있음을 검증
